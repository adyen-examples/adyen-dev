Here is our last newsletter for 2022: blogs, conferences and new releases of our libraries and plugins.

Fun fact, Advent of Code has started! Starting on the 1st and ending on the 25th of December (Christmas), they release a series of small programming puzzles for a variety of skill sets and skill levels that can be solved in any programming language you like. Following the lore, you slowly build a lovely ASCII Christmas tree with every daily challenge that you solve. To keep our minds sharp, some of our engineers at Adyen solve these challenges on a daily basis for fun. We keep track of our own private leaderboard, laugh about our solutions, and challenge you to do the same as well!

We are excited for 2023 as we have recently launched our brand new YouTube channel to help our developers! We will be adding content over time, so feel free to let us know what you would like to see on Twitter!

Latest Releases
Adyen Web 5.30.0 
  • Added new payment method: DuitNow.
  • To enable the instant payment button for Google Pay, you can now set  instantPaymentTypes to either googlepay or paywithgoogle.
  • The expiryDate field now includes an icon showing its location.
  • In the CVC/CVV field, you can set digits to appear as dot icons by setting the new maskSecurityCode property to true.
  • The checkoutAttemptId field is now moved to the paymentMethod object
  • For Pix, when showPayButton is set to false, the payment form now appears when the shopper selects Pix from the payment methods list.
  • For Klarna using the /sessions endpoint, you no longer have to implement the onAdditionalDetails callback. Previously, you had to implement the onAdditionalDetails callback to make a request to the correct endpoint.
  • When you create the 3DSecure2 Component with createFromAction, the component no longer hangs if you make more than one transaction of the same type.
Adyen Ruby Api Library 6.2.0 
  • Support customizable Faraday connection options.
  • Updated Modifications API for Checkout v67.
Adyen Android 4.9.0 
  • For cards, in AddressConfiguration.PostalCode mode, you can now specify if the postal code field required.
  • For BCMC, you can now specify if the card holder name field is required.
  • After the card brand is detected and the shopper enters the full card number in the card number input field, the focus will automatically move to the next input field.
  • If you currently set the postal code input field in the AddressConfiguration as .setAddressConfiguration(AddressConfiguration.PostalCode), you must update it to .setAddressConfiguration(AddressConfiguration.PostalCode()).
Adyen iOS 4.10.2 
  • When the shopper submits a payment, the billing address that the shopper entered is now submitted. Previously, when the selected country doesn't include a state or province field, the submitted value was nil.
  • The Drop-in/Component now detects when a shopper enters an invalid email address in the payment form. Previously, sometimes the Drop-in/Component detected invalid email addresses as valid.
  • For cards, when the shopper enters a card number and a dual branded card is detected, a brand is no longer selected.
Adyen Magento2 8.11.0 
  • Added adyen_checkout_frontend_region in StoreConfig type.
  • Added Atome support through line items for Open Invoice function.
  • Added merchantData for Klarna for Open Invoice function.
  • Now fetches CC brands array from payment methods response for multishipping.
  • Fixed Vault token generation for 3DS2 payments.
Adyen Apex Api Library 2.0.0.0 
  • Added new models for Adyen Salesforce B2B Lightning 1.1.0.
  • Added new Payment Modification Models and Interfaces based on Adyen Checkout API.
Adyen Shopware6 3.8.1 
  • Fixed Showpare Affiliate marketing for Adyen Payment Method.
  • Added voucher details to order overview.
  • Added commands to enable or disable Adyen payment methods.
  • Support new GooglePay tx_variant.
  • Support cancellation for OFFER_CLOSED and CANCELLED notifications.
Adyen Salesforce Commerce Cloud 22.2.1 
  • Added new independent Business Manager configuration page.
  • Added native support for UPI payment method.
  • Support gift cards using Order API in SFRA.

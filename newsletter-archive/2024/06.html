React Native Drop-in/Components v2 is now live!

We are thrilled to announce the release of React Native Drop-in/Components v2! This major version release significantly improves the developer experience and reduces the time it takes to integrate and go live with our Checkout product.
Key features:
  • /sessions support: Simplifies the complexity of different payment flows.
  • Expo support: Eases the integration process.
  • Standalone action handling: Ideal for API-only users.
Upgrading from v1 to v2 involves minimal effort due to limited breaking changes. Check out the integration guide to implement React Native Drop-in/Components with the /sessions flow.

Single CA - API credentials and webhooks

For merchants using Adyen for Platforms integrations, starting June 30, you can manage all your API credentials and webhooks within the standard Customer Area. The Balance Platform Customer Area remains accessible, but we recommend using the standard Customer Area to use new features such as API logs for Balance Platform APIs. Future updates will only be available in the standard Customer Area.

Learn more here.

Adobe Commerce v9.6.0 with new iDEAL flow

We have introduced the new iDEAL payment flow in Adobe Commerce v9.6.0. Shoppers are redirected to the iDEAL payment page to select their bank and authorize the payment.
New payment methods:
  • PayPo
  • Online banking India
  • Wallets India
  • UPI
